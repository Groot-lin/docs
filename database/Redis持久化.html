    Redis持久化

    lin 2023/11/20 Redis

    Redis是内存存储,服务重启可能会丢失数据

    所以我们想个办法将Redis的数据做持久化

    # 一.RDB

    全称 Redis Database Backup file, Redis数据备份文件,也被叫做Redis数据快照

    就是把内存中的所有记录都记录磁盘中

    当Redis实例故障重启后,从磁盘读取快照文件,恢复数据

    快照文件被称为RDB文件,默认是保存在当前运行目录

    # 1.RDB使用

    下面是两种执行RDB的命令

    # 1).save 命令

    由Redis主进程来执行RDB,会阻塞所有命令

    如果数据量比较大的话

    且磁盘IO较慢

    会导致等待时间过长

    一般用于快要停机时使用

    故不建议使用

    # 2).bgsave 命令

    开启子进程执行RDB,避免主进程受到影响

    后台保存开始

    # 3).内部触发RDB

    在redis.conf文件内

    #900秒内,如果至少有一个key被修改了,则执行bgsave 
save 900 1

    2

    # 2.fork原理

    fork : 系统调用,从当前进程中切出去一个子进程

    bgsave开始时会fork主进程得到子进程,子进程共享主进程的内存数据

    完成fork后读取内存数据并写入RDB文件

    bgsave操作是几乎零阻塞的

    为什么是几乎?

    因为读取内存数据并写入RDB文件这个操作是异步的

    但是fork操作是阻塞的

    所以我们需要减少fork所花费的时间

    fork原理

    我们知道

    Redis是操作物理内存的

    但是Linux的所有进程都无法直接操作物理内存

    而是给每个进程都分配一个虚拟内存

    主进程只能操作虚拟内存

    Linux会维护一个虚拟内存与物理内存的映射表

    称为页表

    fork时,我们复制的是页表

    有了相同的映射关系

    实现了主进程和子进程的内存空间共享

    再通过子进程写一个新的RDB文件替换旧RDB文件

    但是有个问题

    子进程在写数据时,主进程可以修改数据

    如何保证主进程与子进程之间的数据一致性?

    fork采用的是copy-on-write技术 :

    fork会把当前数据标为read-only只读模式

    • 当主进程执行读操作时,访问共享内存
    • 当主进程执行写操作时,则会拷贝一份数据执行写操作(极端情况下内存翻倍,挤爆服务器)

    RDB的应用场景就是追求更高的启动速度,也就是性能

    # 二.AOF

    AOF全称为Append Only File(追加文件)

    Redis处理的每一个写命令都会记录在AOF文件，可以看做是命令日志文件

    恢复数据时,执行AOF文件,执行文件中的命令

    假设有一个命令

    set num 123
    1

    那么AOF的文件形式就是

    $3
set
$3
num
$3
123
    1
    2
    3
    4
    5
    6

    3表示长度

    AOF默认是关闭的，需要修改redis.conf配置文件来开启AOF:

    #是否开启AOF功能,默认是no
appendonly yes

#AOF文件的名称
appendfilename "appendonly.aof"
    1
    2
    3
    4
    5

    AOF的命令记录的频率也可以通过redis.conf文件来配:

    #表示每执行一次写命令，立即记录到AOF文件
appendfsync always
#写命令执行完先放入AOF缓冲区，然后表示每隔1秒将缓冲区数据写到AOF文件，是默认方案
appendfsync everysec
#写命令执行完先放入AOF缓冲区，由操作系统决定何时将缓冲区内容写回磁盘
appendfsync no

    配置项 刷盘实际 优点 缺点
    Always 同步刷盘 可靠性高 性能影响大
    everysec 每秒刷盘 性能适中 最多丢失一秒数据
    no 操作系统控制 性能最好 可靠性较差

    AOF的问题

    因为是记录命令，AOF文件会比RDB文件大的多

    而且AOF会记录对同一个key的多次写操作，但只有最后一次写操作才有意义

    通过执行bgrewriteaof命令，可以让AOF文件执行重写功能，用最少的命令达到相同效果

    AOF的应用场景就是对数据安全性要求较高
