    JVM内存结构

    lin 2023/9/3 JVM

    以JDK1.8为例

    线程私有的:

    • 程序计数器

    • 虚拟机栈

    • 本地方法栈

    线程共享的:

    • 堆
    • 方法区
    • 直接内存

    # 一.线程私有

    # 1.程序计数器 PC Register

    程序计数器会随着线程的启动而创建

    用来存放下一条JVM指令的执行地址

    物理上使用寄存器来实现

    特点:

    • 线程私有
    • 不会存在内存溢出

    # 2.Java虚拟机栈-JVM Stacks

    # 虚拟机栈

    • 每个线程运行时所需要的内存称为虚拟机栈

    • 栈由一个个栈帧(每个方法运行需要的内存)组成

    • 每个线程只能友一个活动栈帧,对应当前正在执行的那个方法

    # 栈帧包括 :

    • 局部变量表
    • 操作数栈
    • 动态链接
    • 方法返回地址

    # 问题 :

    • 垃圾回收涉不涉及刀栈内存?
      • 不涉及
    • 栈内存设置越大程序跑得越快吗?
      • 不是,内存总大小一定,栈内存越大,线程数越少
    • 方法内的局部变量是否线程安全
      • 局部变脸线程私有,不会受到其他线程干扰,除非时被static修饰,如果局部变量引用了对象,并逃离方法的作用范围,需要考虑线程安全

    # 栈内存溢出-java.lang.StackOverflowError

    修改参数-Xss 1m

    1. 栈帧过多导致内存溢出(如递归未设置终止条件导致溢出)
    2. 栈帧过大直接导致超过栈大小导致溢出(可能性较小)

    # 3.本地方法栈-Native Method Stacks

    Java虚拟机调用本地方法时的运行内存空间

    虚拟机栈为虚拟机执行 Java 方法 （也就是字节码）服务

    而本地方法栈则为虚拟机使用到的 Native 方法服务

    其结构与虚拟机栈类似

    # 二.线程共有

    # 1.堆-Heap

    # 堆

    Java虚拟机所管理的内存中最大的一块

    Java堆是所有线程共享的的一块内存区域

    在虚拟机启动时创建

    # 作用:

    存放对象实例

    使用new关键字创建的

    # 堆内存溢出-OutOfMemoryError

    修改参数-Xmx 8m

    命令 :

    Terminal-local里面的命令行

    • jps : 获取当前正在运行的进程及id
    • jmap -heap xxxxx : 检查进程堆内存占用情况

    # 2.方法区 Method Area

    方法区在JVM启动时创建

    方法区是JVM运行时数据区域的一块逻辑区域

    方法区会存储已被虚拟机加载的 类信息、字段信息、方法信息、常量、静态变量、即时编译器编译后的代码缓存等数据。

    # JDK1.8以前

    方法区在线程共享中

    设置最大内存-XX:MaxPermSize=8m

    内存溢出报错

    java.lang.OutOfMemoryError: PermGen space
    1

    # JDK1.8

    JDK1.8以后方法区改名元空间放在本地内存中

    设置最大内存-XX:MaxMetaspaceSize=8m

    # 三.本地内存

    以上是运行时数据区域

    下面是本地内存

    # 运行时常量池

    存放在方法区中

    包括

    • 字面量 : 源代码中的固定值表示法,即通过字面我们就能知道其值的含义(整数,浮点数,字符串等)
    • 常量池表,虚拟机指令根据这张常量表找到要执行的类名,方法名,参数类型,字面量等信息

    当某类被加载,它的常量池信息就会放入运行时常量池,并把里面的符号地址变为真实地址

    # 字符串常量池-StringTable

    字符串常量池是JVM为为了提升性能和减少内存消耗,正对字符串(String 类)专门开辟的一块区域

    主要目的是避免字符串的重复创建

    例：

    String str = "abc";
    1
    1. 将字符串对象"abc"的引用保存在字符串常量池中
    2. 直接返回字符串常量池中的字符串对象"ab"的引用

    # 直接内存

    一种特殊的内存缓冲区,属于操作系统内存

    磁盘文件可以读到直接内存

    Java堆内存也可也直接访问

    减少了一次缓冲操作
