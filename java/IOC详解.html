    # 一.谈谈Ioc

    Ioc即 Inversion of Control

    翻译为 '控制反转'

    就是把复杂系统分解成相互合作的对象，这些对象类通过封装以后，内部实现对外部是透明的，从而降低了解决问题的复杂度，而且可以灵活地被重用和扩展

    简单来说就是将创建和管理Bean(即Java对象)的工作交给Spring容器

    这就是控制反转,将获得依赖对象的过程由自身管理变为了由IOC容器主动注入

    如

    正转 :

    Object object = new Object();
    1

    反转 :

    @Autowired
private Object object;
    1
    2

    Ioc也叫依赖注入

    由IOC容器在运行期间，动态地将某种依赖关系注入到对象之中

    如

    普通对象 :

    对象 --- 无依赖 ---> null

    Spring对象

    对象 --- 依赖 --->容器

    # 1.BeanFactory

    BeanFactory是一个用来访问 Spring 容器的 root 接口

    要访问 Spring 容器，我们将使用 Spring 依赖注入功能，使用 BeanFactory 接口和它的子接口

    主要功能为

    • 定义Spring容器的基本功能
    • 负责管理Bean的生命周期
    • Bean的实例化和管理
    • 依赖注入

    问 : BeanFactory 和 FactoryBean 的区别?

    答 :

    BeanFactory是Ioc容器的重要实现接口,负责生产和管理Bean

    FactoryBean可以说为IOC容器中Bean的实现提供了更加灵活的方式,可以在getObject()方法中灵活配置

    # 2.ApplicationContext

    ApplicationContext是对BeanFactory的一个拓展

    有四种拓展的功能分别对应多实现的四个接口

    1. MessageSource，提供国际化支持
    2. 资源访问，比如 URL 和文件
    3. 事件传递。通过实现ApplicationContextAware接口
    4. 载入多个配置文件

    # 三.后处理器

    Spring后处理器是SPring对外开发的重要扩展点,允许开发者介入到Bean的整个实例化流程中来

    以达到动态注册,修改BeanDefinition及Bean的作用

    Spring主要有两种后处理器:

    • BeanFactoryPostProcessor : Bean工厂后处理器,在BeanDefinitionMap填充完毕,Bean实例化之前执行
    • BeanPostProcessor : Bean后处理器,一般在Bean实例化之后,填充到单例池singletonObjects之前执行

    我们从BeanFactory的实现说起

    public class TestBeanFactory {

    public static void main(String[] args) {
        DefaultListableBeanFactory beanFactory = new DefaultListableBeanFactory();
        // bean 的定义（class, scope, 初始化, 销毁）
        AbstractBeanDefinition beanDefinition =
        BeanDefinitionBuilder.genericBeanDefinition(Config.class).setScope("singleton").getBeanDefinition();
        beanFactory.registerBeanDefinition("config", beanDefinition);

      

        for (String name : beanFactory.getBeanDefinitionNames()) {
            System.out.println(name);
        }

        beanFactory.preInstantiateSingletons(); // 准备好所有单例
        System.out.println(">>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>> ");
//        System.out.println(beanFactory.getBean(Bean1.class).getBean2());
        System.out.println(beanFactory.getBean(Bean1.class).getInter());
  
    }

            return bean2;

        @Autowired

    static class Bean2 {

    BeanFactory的最重要的一个实现,它根据bean的定义去创建bean

    # 1.BeanFactory后处理器

    但是DefaultListableBeanFactory 定义好以后其实并没有生效,并未识别到@Configuration

    所以需要注册BeanFactory后处理器

    后处理器是对BeanFactory的一个扩展,提供对注解的解析功能

    原始的BeanFactory并没有提供解析注解的功能

    我们使用以下代码给BeanFactory增加后处理器

    // 给 BeanFactory 添加一些常用的后处理器
AnnotationConfigUtils.registerAnnotationConfigProcessors(beanFactory);

// BeanFactory 后处理器主要功能，补充了一些 bean 定义
beanFactory.getBeansOfType(BeanFactoryPostProcessor.class).values().forEach(beanFactoryPostProcessor -> {
            beanFactoryPostProcessor.postProcessBeanFactory(beanFactory);
        });

    # 2.Bean后处理器

    添加了之后可以识别Coniguration,但是却无法构造出Bean1

    因为目前BeanFactory无法识别@Autowired

    所以得注册Bean后处理器

    Bean后处理器针对bean的生命周期的各个阶段提供扩展

    // Bean 后处理器, 针对 bean 的生命周期的各个阶段提供扩展, 例如 @Autowired @Resource ...
beanFactory.getBeansOfType(BeanPostProcessor.class).values().stream()
           .sorted(beanFactory.getDependencyComparator())
           .forEach(beanPostProcessor -> {
               System.out.println(">>>>" + beanPostProcessor);
               beanFactory.addBeanPostProcessor(beanPostProcessor);
        });

    ClassPathXmlApplicationContext是ApplicationContext的实现类

    用于xml配置Bean的方式来读取bean的信息

    其原理就是

    由XmlBeanDefinitionReader的对象去读取xml中bean的信息,而后加载到beanFactory中

    		DefaultListableBeanFactory beanFactory = new DefaultListableBeanFactory();
        System.out.println("读取之前...");
        for (String name : beanFactory.getBeanDefinitionNames()) {
            System.out.println(name);
        }

        System.out.println("读取之后...");
        //读取a02中的bean信息,XmlBeanDefinitionReader用于读取
        XmlBeanDefinitionReader reader = new XmlBeanDefinitionReader(beanFactory);
        reader.loadBeanDefinitions(new ClassPathResource("a02.xml"));
        for (String name : beanFactory.getBeanDefinitionNames()) {
            System.out.println(name);
        }

    # 五.IOC工作流程

    话又说话来了

    IOC的工作流程是什么呢?

    主要可以分为两个阶段:

    # 1.IOC容器初始化

    这个阶段主要是根据程序里面定义的xml

    或者注解等Bean的声明方式

    通过解析和加载后生成BeanDefinition实体

    包含Bean的一些定义和属性

    然后将BeanDefinition注册到IOC容器中

    具体来说就是放到一个Map集合中

    由IOC容器来维护这些信息

    # 2.Bean的初始化和依赖注入

    通过反射针对没有设置lazy-init属性的的单例bean

    接着就是完成bean的依赖注入
