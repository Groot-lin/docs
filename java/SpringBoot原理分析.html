    SpringBoot原理分析

    lin 2023/11/17 Spring

    SpringBoot两大功能

    一个是起步依赖

    能帮助你管理好依赖的信息

    二个是自动配置Bean

    SpringBoot如何获取Bean,这是个问题

    下面我们详细说说

    # 一.起步依赖

    起步依赖有两个重要的

    spring-boot-starter-parent

    SpringBoot工程需要引用的父依赖

    根据源码我们发现

    spring-boot-starter-parent还依赖于另外一个父工程spring-boot-dependencies

    而在spring-boot-dependencies中融合了各个技术的版本

    这样就防止开发人员在开发时遇到版本冲突问题

    所以说Spring只是帮你做了各个技术实现的整合,是站在Spring的肩膀上

    spring-boot-starter-web

    spring-boot-starter-web依赖启动器的主要作用是提供Web开发场景所需的底层所有依赖

    比如

    Spring-web

    Spring-webmvc

    所以项目间接依赖于这些底层依赖

    这就是依赖传递

    # 二.自动配置

    # 1.@Conditional

    是在Spring4.0增加的条件判断功能

    通过这个功能可以实现选择性创建bean操作

    Spring是如何知道要创建哪个bean的?

    Spring会用Condition判断当前容器环境中是否有需要获取的Bean

    @Conditional(Class<T>)
//即Conditional(xxx.class)
    1
    2

    Class< T>这个类就是一个是否创建Bean的开关

    该类的 **matches()**中设定条件改变返回值

    返回true则创建

    返回false则创建

    SpringBoot中给我们定义了一堆Condition包

    可以根据各种条件是否创建一个bean

    例如

    • 31行 : 要创建这个bean方法先满足redisTemplate这个Bean未被创建

    # 2.切换内置web服务器

    总所周知

    SpringBoot的web环境中默认使用tomcat作为内置服务器,存放在spring-boot-starter-web依赖中

    但是其实SpringBioot提供了4中内置服务器供我们选择

    具体的切换方法是在spring-boot-starter-web依赖中排出一个

    引入另一个

    # 3.@Enable*注解原理

    SpringBoot中提供了很多Enable开头的注解，这些注解都是用于动态启用某些功能的

    而其底层原理是使用@Import注解导入一些配置类，实现Bean的动态加载

    SpringBoot工程是否可以直接获取jar包里面的Bean?

    答案是不可以

    为什么?

    我们看看@SpringBootApplication的源代码

    @ComponentScan这个注解扫描的是启动类当前包以及子包

    所以加载不到

    但是我们有两种方式解决 :

    1. @ComponentScan注解扫描Bean所在包

    2. @Import()该bean的class,如 @Import(xxx.class)

    3. 对Import进行封装

    # 4.@Import注解

    有四种用法 :

    1. 导入Bean,@Import(xxx.class)
    2. 导入配置类,@Import(xxxConfig.class)
    3. 导入ImportSelector的实现类
    4. 导入ImportBeanDefinitionRegisterar实现类

    我们进入**@EnableAutoConfiguration**注解

    AutoConfigurationImportSelector就是ImportSelector的实现类对象

    # 5.自动配置流程

    了解了以上内容

    我们来说说SpringBoot的自动配置流程

    1. @EnableAutoConfiguration注解内部使用@Import (AutoConfigurationImportselector.class>)来加载配置类

    2. 配置文件位置:META-INF/spring.factories，该配置文件中定义了大量的配置类，当SpringBoot应用启动时，会自动加载这些配置类，初始化Bean

    3. 但并不是所有Bean都会被初始化,在配置类中使用Confition来加载满足条件的Bean
