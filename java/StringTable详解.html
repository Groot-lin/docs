    StringTable详解

    lin 2023/9/23 JVM

    # 一.字符串常量池

    字符串池在JDK1.7之后存在于堆中的一块区域

    常量池中的信息都会被加载到运行时常量池中,这是 a b ab都是常量池中的符号,还没有变为Java字符串对象

    # 二.示例详解

    String s1 = "a";
    String s2 = "b";
    String s3 = "ab";

    # 3.String s6="ab"和String s7=new String("ab")有什么区别?

    # 1).String s6="ab"直接赋值

    如果字符串常量池中已经存在"ab"

    则会直接返回其引用

    不存在则会创建返回

    所以该方法会创建0个或1个对象

    # 2).String s7=new String("ab")构造方法

    1. 首先会在堆内存中创建一个String对象,值为"ab"

    2. 如果在字符串在字符串常量池中不存在,也会在字符串常量池中创建

    3. 返回堆中对象的引用

    该方法会创建1个或两个对象

    # 四.内存结构

    JDK1.6存放在常量池中,常量池存放在永久代中

    JDK1.7存放在堆内存中

    原因:

    存放在永久代中GC效率太低难以回收,造成大量内存占用导致浪费

    # 五.性能调优

    # 1. 调整桶个数

    StringTable本质是hashtable

    调优在hashable的桶的个数上进行增减

    使用-XX: StringTableSize=200000进行调整

    如果系统中的字符串常量较多,建议增加Hashtable的桶个数

    # 2.考虑是否将字符串对象入池

    使用 **intern()**方法

    可以将重复的字符串执行**intern()**方法,减少入池的字符串
