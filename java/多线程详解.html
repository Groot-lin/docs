    # 一.创建线程的方式

    # 1. 继承Thread

    1. 类继承Thread类
    2. 重写 run() 方法
    3. 创建子类的对象,并启动线程
    //第一步
public class Thread1 extends Thread{
    public static void main(String[] args) {

        //第三步
        Thread1 t1 = new Thread1();
        t1.start();
    }

    //第二步
    @Override
    public void run() {
        for(int i = 0;i<100;i++)
            System.out.println(i+1);
    }
}

    # 2.实现Runable接口

    1. 实现Runable接口
    2. 重写run方法
    3. 创建自己类的对象
    4. 创建Thread类对象,并开启线程
    public class Thread2 implements Runnable{
    public static void main(String[] args) {
        //创建自己的类的对象
        Thread2 r = new Thread2();

        //创建线程对并传入参数
        Thread t1 = new Thread(r);
        Thread t2 = new Thread(r);

        t1.setName("线程1");
        t2.setName("线程2");

        t1.start();
        t2.start();
    }
    @Override
    public void run() {
        Thread t = Thread.currentThread();
        for (int i = 0;i<100;i++){
            System.out.println(t.getName()+" "+(i+1));
        }
    }
}

    # 3. 实现Callable接口

    特点 : 可以获取到多线程运行的结果

    1. 创建一个类实现Callable接口

    2. 重写call (与run()方法不同的是该方法有返回结果)

    3. 创建类对象

    4. 创建FutureTask对象(作用管理多线程运行的结果)

    5. 创建Thread类的对象,并启动

    public class Thread3 implements Callable<Integer> {
    public static void main(String[] args) throws ExecutionException, InterruptedException {

        //创建类对象
        Thread3 t3 = new Thread3();
        
        //创建FutureTask对象(管理多线程运行的结果)
        FutureTask<Integer> ft = new FutureTask<>(t3);
        
        //创建Thread类的对象,并启动
        Thread t = new Thread(ft);
        t.start();


        //获取结果
        System.out.println(ft.get());
    }
    @Override
    public Integer call() throws Exception {
        //求1~100的和
        int sum = 0;
         for (int i = 1;i<=100;i++){
             sum += i;
         }
        return sum;
    }
}

    # 二.多线程常用方法

    方法名称 说明
    String getName() 返回此线程的名称
    void setName(String Name) 设置线程名称
    static Thread currentThread() 获取当前线程对象
    static void sleep(long time) 休眠指定时间,单位毫秒
    setPriority( int newPriority) 设置线程优先级
    final int getPriority 获取线程优先级
    final void setDaemon(boolean on) 设置为守护线程
    public static void yield() 礼让线程
    public static void join() 插入线程

    # 三.线程优先级

    • 最低优先级 Thread.MIN_PRIORITY = 1
    • 最高优先级 Thread.MAX_PRIORITY = 10
    • 普通优先级 Thread.NORM_PRIORITY = 5 (默认)

    # 1. CPU调度 :

    • 抢占式调度 : 各个线程抢占cpu资源,随机执行
    • 非抢占式调度 : 各个线程按顺序轮流执行

    优先级越高,抢占到cpu资源的概率越高

    # 2. 守护线程

    当其他非守护线程结束时

    守护线程 陆续 结束

    应用场景 :

    聊天 : 线程1

    传输文件 : 线程2

    当聊天窗口关闭即结束时,传输文件应该陆续结束

    故传输文件可以设置为守护线程

    # 3.礼让线程

    Thread.yield();

    出让当前CPU的执行权

    # 4.插队线程

    Thread.join();

    # 四.生命周期

    1. 新建 ---- 创建线程对象
    2. 就绪 --- 抢CPU的执行权
      • 抢到 --- 执行3
      • 未抢到 --- 执行2
    3. 运行 --- 运行线程程序
    4. 阻塞/等待/计时等待 --- 运行时的代码遇到sleep方法或其他阻塞方法,继续回到2
    5. 死亡 --- 执行完毕

    # 六.等待唤醒机制

    就是在一个线程进行规定操作后，就进入等待状态waiting，

    等待其他线程执行完他们的指定代码过后再将其唤醒notify

    常见方法

    方法名称 说明
    void wait() 等待
    void notify() 唤醒
    void notifyAll() 唤醒所有

    调用方法时使用锁对象去调用方法

    # 七.线程池

    # 1.原理

    1. 创建一个池子,池子中是空的
    2. 提交任务时,池子会创建新的线程对象,任务执行完毕,线程归还给池子.下次再次提交任务时,不需要创建新的线程,直接复用已有的线程即可
    3. 所有任务全部执行完毕,关闭线程池
    // 1.获取线程池对象
        ExecutorService pool = Executors.newCachedThreadPool();

        // 2.提交任务
        pool.submit(new Thread2());
        Thread.sleep(1000);
        pool.submit(new Thread2());
        Thread.sleep(1000);
        pool.submit(new Thread2());
        Thread.sleep(1000);
        pool.submit(new Thread2());
        Thread.sleep(1000);
        pool.submit(new Thread2());

        //3.销毁线程池
        pool.shutdown();

    # 2.自定义线程池

    生产中一般不使用Executors获取线程池对象

    使用ThreadPoolExecutors类自定义参数

    参数 说明
    核心线程数量 不能小于零
    最大线程数 >= 核心线程数
    空闲线程最大存活时间 值
    空闲线程最大存活时间 单位
    任务队列
    创建线程工厂 线程池如何获取线程
    任务的拒绝策略 默认凡是为丢弃并抛异常

    例

    核心线程数 --- 3

    最大线程数 --- 6

    任务队列 --- 3

    这时候有10个任务

    任务1~3被核心线程处理

    任务4~6存放在任务队列

    任务7~9被临时线程处理

    任务10执行拒绝策略
