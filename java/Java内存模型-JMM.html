    Java内存模型-JMM

    lin 2023/9/30 JUC

    JMM规定的是一个规范，通过规范定义了程序中各个变量的访问方式

    围绕着原子性、有序性、可见性三点展开

    # 一.原子性

    # 1.问题

    两个线程对静态变量ｉ进行分别自增5000次和自减5000次

    结果不一定为0

    # 2.分析

    在JMM中,完成静态变量的自增自减需要在主存和线程内存中进行数据交换

    # 3.解决

    Java保证原子性-synchronized

    语法

    synchronized(唯一对象){
    原子性操作;
}

    运行以上代码,mian线程对静态变量run的修改会导致t线程并不可见,从而使t线程一直运行

    # 2.分析

    1. 初始状态,他线程刚开始从主内存读取了run的值到工作内存
    2. 因为t线程要频繁从主内存中读取run的值,JIT编译器会将run的值缓存至自己工作内存中的告诉缓存中,减少对主内存中run的访问,提高效率
    3. 1秒之后,main1线程修改了run的值,并同步到主存,而t是从自己工作内存中的高速缓存中读取这个变量的值,永远是旧值

    # 3.解决

    在变量前加上volatile关键字,适合一个写线程,多个读线程

    特性

    • 不保证原子性
    • 保证可见性
    • 禁止指令重排

    # 2.分析

    I_Result是一个对象，有一个属性r1用来保存结果，问，可能的结果有几种? 情况如下:

    • 情况1:线程1先执行，这时ready = false，所以进入else分支结果为1
    • 情况2∶线程2先执行num=2，但没来得及执行ready = true，线程1执行，还是进入 else分支，结果为1
    • 情况3∶线程2执行到ready = true，线程1执行，这回进入if分支，结果为4(因为mum已经执行过了)

    但还有一种情况是0 这种情况下是:线程2执行ready = true，切换到线程1，进入 if分支，相加为o，再切回线程2执行num=2

    这种情况叫指令重排
