    # 一.AOP是什么

    AOP, 即 Aspect Oriented Programming

    面向切面编程,是面向对象编程OOP的升华

    横向对不同事物的抽象

    比如属性的属性,方法的方法,对象的对象都可以组成一个切面

    这么说有点抽象

    本人的理解就是对原来的属性方法对象做增强扩展操作

    # 1.AOP思想的实现方案

    动态代理技术,在运行期间,对目标对象的方法进行增强

    代理对象同名方法内可以执行原有逻辑的同时

    嵌入其他增强逻辑或其他对象的方法

    假设有一个类A

    public class A{
    methodA1(){}
    methodA2(){}
    methodA3(){}
}

    经典的AOP应用有 :

    • 日志记录
    • 事务管理
    • 权限验证
    • 性能检测

    # 2.核心概念

    专业术语

    名称 说明
    Target 目标对象 被增强的对象
    Proxy 代理对象 增强后的对象
    Joinpoint 连接点 目标对象中可以被增强的方法
    Pointcut 切入点 目标对象中实际被增强的方法
    Advice通知 增强部分的代码逻辑
    Aspect切面 增强+切入点
    Weaving织入 通知和切入点动态组合的过程

    通知类型

    通知 说明
    前置通知 通知方法在目标调用之前执行
    后置通知 通知方法在目标方法返回或异常后调用
    最终通知 都会执行
    异常通知 通知方法会在目标方法抛出异常后调用
    环绕通知 前后调用

    # 二.原理剖析

    Spring如何生成Proxy代理对象?

    # 1.xml方式

    1. 首先进行自定义命名空间解析
    <aop:config>
    1
    1. 为该标签(config)设置一个parse解析器
    2. 解析器为Spring容器中注入AspectJAwareAdvisorAutoProxyCreator,本质是一个BeanPostProcessor,bean后处理器
    3. 在AspectJAwareAdvisorAutoProxyCreator的after方法中生成Bean代理对象

    # 2.注解方式

    1. 首先对注解进行解析
    @EnableAspectJAutoProxy
    1

    1. 该注解往Spring容器中添加了一个后处理器,名为AnnotationAwareAspectJAutoProxyCreator

    2. AnnotationAwareAspectJAutoProxyCreator的after方法中生成了Bean的代理对象
