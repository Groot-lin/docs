    # 一.IPv4地址

    网际协议IP是TCP/IP协议簇中两个主要的协议质疑

    IPv4地址是给因特网上的每一台主机或者路由器的每一个接口分配一个在全世界范围内是唯一的32比特的标识符

    可以简单理解为一台主机的ID号

    # 点分十进制表示

    由于32比特的IPv4地址不方便阅读,记录

    所以将32位的比特分成四部分

    每部分8比特分为一组

    写出对应十进制数

    假设有这么一串IPv4地址

    00001010 11110000 00001111 10101010

    其点分十进制为10.240.15.170

    # 二.IPv4地址的发展

    # 1.分类IP地址

    互联网早期使用的是分类的IP地址

    将32位的IP地址改变网络号和主机号

    划分成ABCDE五类地址

    只要ABC类地址可以分配给网络中的主机或路由器接口

    在这里面

    主机号"全0"的地址是网络地址

    主机号"全1"的地址是广播地址

    不能分配给主机或者路由器接口

    # 2.划分子网的IP地址

    分类IP网络号固定

    后来研究出更加灵活的划分子网

    即在主机号中插入一个子网号

    将两级IP划分为三级

    但是随着互联网的迅猛发展

    该方案被代替

    # 3.无分类编制CIDR

    这种方式摒弃了固定的网络号和主机号

    而是选择x位的网络号

    CIDR的表示形式为a.b.c.d/x

    x范围是0~32,表示前面有x位是网络号

    如:10.100.122.2/24

    所有网络前缀相同的就组成了一个地址块

    看看几种情况:

    • 128.14.32.7

      这是一个IP,但不知道网络号

    • 128.14.32.7/20

      这是一个IP,网络前缀是20位

    • 128.14.32.0/20

      这是一个地址块,包含多个IP地址,代表一类

    # 4.地址掩码

    实际表示CIDR时

    总不可能在后面加上斜杠和数字吧

    要计算出网络地址,需要地址掩码

    地址掩码有一串1和一串0组成

    其中1的个数就是网络前缀的长度,又称子网掩码

    例:

    有一个IP地址128.14.32.7/20

    那么它的子网掩码就是20个1和12个0

    11111111 11111111 11110000 00000000

    将IP与掩码按位相与(AND)

    得到网络号为128.14.32.0
