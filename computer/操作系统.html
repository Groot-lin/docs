    操作系统

    lin 2023/12/28 操作系统

    # 一.什么是操作系统?

    总所周知

    操作系统是计算机的基石

    相当于人的头部

    来看看操作系统的定义:

    操作系统是一组控制与管理计算机软硬件资源,合理地对各类作业进行调节,一级方便用户使用的程序的集合

    # 二.操作系统的主要功能

    # 1.处理器管理

    对中央处理器,也就是CPU的管理

    多道操作系统下,CPU的分配与运行以进程为单位

    所以可以归结为对进程和线程的调度管理

    # 2.存储管理

    即对内存空间的管理

    程序需要运行就涉及到将程序从外存调到内存中

    存储管理的主要任务就是为每道程序/作业分配内存

    包括:内存分配,内存保护,地址变换和内存扩充

    # 3.设备管理

    指管理计算机处理CPU以及内存之外的所有输入输出设备

    如I/O设备

    首要任务就是为这些设备提供驱动程序或者控制程序

    并屏蔽细节

    # 4.文件管理

    之前提到的都是硬件资源

    操作系统还需要管理软件资源

    也就是文件

    程序与数据以为文件形式存放在外存,需要运行时才装入内存

    主要功能包括:文件存储空间管理,文件目录管理,文件操作等

    # 5.网络通信管理

    现代操作系统需要为用户提供用户便捷可靠的网络通信服务

    包括:网络资源管理,数据通信管理,网络管理的

    注:不是操作系统的重点

    # 三.用户态与内核态

    操作系统的可靠性和安全性一旦出现问题

    会造成整个系统的崩溃,影响用户程序

    解决方法就是将进程在系统上的运行分为两个级别:

    • 用户态

    用户态运行的进程可以直接读取用户程序的数据，拥有较低的权限

    一般来说,一个程序能运行在用户态就让他运行在用户态

    • 内核态

    核态运行的进程几乎可以访问计算机的任何资源包括系统的内存空间、设备、驱动程序等

    不受限制，拥有非常高的权限

    # 1.为什么需要分两种状态

    在 CPU 的所有指令中，有一些指令(特权指令)是比较危险的比如内存分配、设置时钟、IO 处理等

    如果所有的程序都能使用这些指令的话，会对系统的正常运行造成灾难性地影响,降低系统的安全性

    # 2.用户态与内核态之间如何切换

    一般有三种方式:

    1. 系统调用

      用户态进程主动要求切换到内核态的一种方式

    2. 中断

      当外围设备完成用户请求的操作后，会向 CPU 发出相应的中断信号，这时 CPU 会暂停执行下一条即将要执行的指令转而去执行与中断信号对应的处理程序

    3. 异常

      当 CPU 在执行运行在用户态下的程序时，发生了某些事先不可知的异常，这时会触发由当前运行进程切换到处理此异常的内核相关程序中，也就转到了内核态

    # 3.什么是系统调用

    操作系统内核中设置了一组用于实现各种系统功能的子程序

    用户程序在用户态执行中需要系统内核提供服务时能够使用这组子程序

    说人话就是

    就是用户态程序调用内核态程序
