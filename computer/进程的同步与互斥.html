    进程的同步与互斥

    lin 2024/1/1 操作系统

    先说几个老生常谈的概念

    # 1.并行与并发

    并发:

    一个处理器同时处理多个任务,看起来同时进行,实则切换得很快,看不出来只有一个进行

    并行:

    多个处理器同时处理任务

    # 2.同步与互斥

    互斥:

    多个并发进程中,某个进程访问临界区资源时,阻塞其他进程

    同步:

    多个并发进程中,在某个节点需要相互等待与互通消息

    这种相互等待对方协调消息得关系就叫同步

    操作系统中如何实现同步与互斥这种写协作关系呢?

    答案:

    • 锁
    • 信号量

    # 一.信号量

    # 1.PV操作

    总结一下PV操作

    若存在信号量S

    • P操作

      申请资源

      信号量减一

      若减一以后S<0

      说明资源不足

      阻塞当前进程

    • V操作

      释放资源

      信号量加一

      若加一以后S<=0

      说明阻塞队列中有进程需要该资源

      唤醒阻塞队列中的一个资源

    现在我们来看看信号量机制是如何解决进程协作的

    # 2.生产者-消费者问题

    问题可以抽象成:

    生产者向消费者提供产品

    共享一个有界缓冲池

    消费者取出产品消费

    我们需要三个信号量:

    • 互斥信号量mutex:用于互斥访问缓冲区,初始值为1
    • 资源信号量full:满缓冲区的数量,初始值为0
    • 资源信号量empty:满缓冲区的数量,初始值为1

    代码如下:

    # 3.哲学家进餐问题

    问题描述:

    五个哲学家,中间有五只筷子(看起来是中国的哲学家)

    哲学家围在一起先思考，思考中途饿了就会想进餐,也就是拿到两只筷子才进餐

    吃完后把两只筷子放回

    问题来了

    如何保证哲 学家们的动作有序进行，而不会出现有人永远拿不到筷子呢？

    # 第一种方案

    # define N 5                //哲学家个数
semaphore fork[5];          //表示筷子

void smart_person(int i){
    while(true){
        think();            //思考
        P(fork[i]);         //拿到左边的筷子
        P(fork[(i+1)%N]);   //拿到右边的筷子
        eat();              //进餐
        V(fork[i]);			//放下左边的筷子
        V(fork[(i+1)%N]);   //放下右边的筷子
    }
}

    # 4.读者写者问题

    该问题就是有一个数据文件可以被多个进程共享,各个进程共享数据文件的方式不同

    • 读者:只能读取数据
    • 写者:既可以读取数据,也能写入数据

    有如下情况

    • 读 - 读 : 同一时刻允许多个读者同时读
    • 读 - 写 : 没有写者时读者才能读，没有读者时写者才能写
    • 写 - 写 : 没有其他写者时，写者才能写

    # 第一种方案

    semaphore mutex = 1;//读写互斥信号量
int count = 0;//正在读操作的个数
semaphore countmutex = 1;//count的互斥修改

//写进程
void writer(){
	while(true){
        P(mutex);
        write();
        V(mutex);
    }
}

//读进程
void reader(){
    while(true){
        P(countmutex); //进入临界区
        if(count == 0){
            P(mutex);
        }
        count++;//读者计数+1
        V(countmutex);
        
        read();//读数据
        
        P(countmutex);
        count--;//读者计数-1
        if(count==0){
            V(mutex);//最后一个读者离开,唤醒写者
        }
        V(countmutex);
    }
}

    # 二.锁

    锁如何实现同步与互斥呢?

    下面来聊聊

    # 1.简单的锁实现

    任何想进入临界区的线程，必须先执行加锁操作

    若加锁操作顺利通过，则线程可进入临界区

    在完成对临界资源的访问后再执行解锁操作，以释放该临界资源。

    (图片来源于小林coding (opens new window))

    大概实现如图

    # 2.死锁的条件

    死锁就是两个线程互相等待对方释放锁

    发生条件如下:

    # I.互斥条件

    多个线程不能同时使用一个资源

    也就是说其他线程获取资源被阻塞

    # II.请求和保持条件

    一个进程请求资源得不到满足而阻塞自己

    # III.不可抢占条件

    进程所获得的资源在未使用完毕前不能被其他进程抢占

    # IV.循环等待条件

    多个进程形成一个循环等待链

    链中每个进程都在等待该链下一个进程所占用的资源
