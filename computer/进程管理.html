    # 进程管理

    就是CPU管理

    将CPU资源分配内XX进程

    本质上是通过改变CPU的指向去调用进程

    所以,操作系统对处理器的管理

    可以简化成对进程的管理

    # 一.进程

    什么是进程和线程就不说了

    书上都有

    说几个重要的

    # 1.线程的控制结构

    包括这几个东西:

    • 程序段:进程包含的指令集合
    • 数据段:进程私有的数据
    • PCB:程序控制块,包含进程的具体信息

    # 2.PCB

    程序控制块,包含挺多重要信息:

    • 进程描述信息

      包括进程id和用户id

    • 进程控制信息

      包括进程当前状态以及优先级

    • CPU相关信息

    • 资源分配清单

    我们之前说

    对CPU的管理本质是对进程的管理

    那么对进程的管理本质就是对PCB的管理

    PCB通过一个双向链表互相连接,找到下一个进程

    # 3.进程的上下文切换

    一个进程切换到另一个进程就叫上下文切换

    在切换过程中,需要将旧进程的现场信息(CPU寄存器和PC)进行一个保存

    以便待会接着进行

    然后加载新的进程

    # 4.进程的五态模型

    如图为进程的五态模型:

    • 创建

      创建完成后进入就绪队列,就绪状态

    • 就绪

      CPU调度后变为运行状态

      也可能因为时间片到了从运行状态到就绪状态

    • 运行

      完成任务进入销毁状态

      时间片到进入就绪状态

      IO事件或者等待资源进入阻塞状态

    • 阻塞

      IO或者等待资源结束后进入就绪状态

    • 销毁

      进程生命结束

    # 二.线程

    一个进程可以创建多个线程

    一个线程只属于一个进程

    # 1.线程进程的联系与区别

    进程是操作系统资源分配的基本单位，而线程是处理器任务调度和执行的基本单位

    最大的区别就是进程拥有独立的4G的虚拟地址空间

    用来装寄存器与栈

    # 2.为什么引入线程

    线程可以比作是轻量级的进程，是程序执行的最小单位,线程间的切换和调度的成本远远小于进程

    而且

    多线程并发编程正是开发高并发系统的基础

    # 3.线程的实现

    包括

    • 用户线程：在用户空间实现的线程，不是由内核管理的线程，是由用户态的线程库来完成线程的管理；
    • 内核线程：在内核中实现的线程，是由内核管理的线程

    # 三.进程调度

    进程调度算法就简单介绍一下吧

    进程调度的衡量标准为周转时间

    计算方式如下:

    • 完成时间-提交时间

    • 运行时间+等待时间

    介绍一下调度算法

    1. 先来先服务算法FCFS

      谁先来谁就先运行

    2. 短作业优先算法SJF

      选择估计运行时间最短的进程执行

    3. 时间片轮转算法RR

      在先来先服务的基础上

      以时间片为基础调度

    4. 高响应比调度算法HRRF

      响应比 = 1 + 等待时间/运行时间

      每次调度计算响应比,将高响应比进程占用CPU
