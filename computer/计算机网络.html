    # 计算机网络

    # 一.计算机网络基础

    # 定义

    计算机网络是指

    将跨区域的独立计算机,通过通信线路连接起来从而实现资源共享的计算机系统

    # 组成

    $$ 组成 \begin{cases} 资源子网 : 提供共享的软件资源和硬件资源 \ \ 通信子网 : 提供信息交换的网络结点和通信线路 \end{cases} $$

    # OSI七层模型

    OSI 七层模型 是国际标准化组织提出一个网络分层模型

    • 应用层 : 为计算机用户提供服务
    • 表示层 : 数据处理,如加密解密,压缩解压
    • 会话层 : 管理应用程序之间的会话
    • 传输层 : 为两台主机进程之间的通信提供通用的数据传输服务
    • 网络层 : 路由和寻址
    • 数据链路层 : 帧编码与误差纠正控制
    • 物理层 : 为数据段设备被提供传送数据通路

    # OSI应用层常用协议

    • Http : 超文本传输协议

    • SMTP : 简单邮件发送协议

    • POP3/IMAP : 邮件接收协议

    • FTP : 文件传输协议

    • Telnet : 远程登录协议

    • SSH : 安全网络传输协议

    • RTR : 实时传输协议

    • DNS : 域名管理系统

    # OSI传输层协议

    • TCP : 传输控制协议

    TCP是面向连接得到,提供全双工的

    • UDP : 用户数据协议

    # OSI网络层协议

    • IP: 网际协议
    • ARP : 地址解析协议
    • ICMP : 互联网控制报文协议
    • NAT : 网络地址转换协议
    • OSPF : 开放式最短路径优先
    • RIP : 路由信息协议
    • BGP : 边界网关协议

    # 二.TCP/IP四层模型---!!!

    TCP/IP 四层模型 是目前被广泛采用的一种模型,我们可以将 TCP / IP 模型看作是 OSI 七层模型的精简版本

    包括 $$ TCP/IP \begin{cases} 应用层\ \ 传输层\ \ 网络层\ \ 网络接口层 \end{cases} $$

    • 应用层:

    提供两个终端设备上的的应用程序之间信息交换服务

    它定义了信息交换的格式,消息会交给下一层来传输

    • 传输层 :

    主要任务就是负责向两台终端设备进程之间的通信提供通用的数据传输服务(数据传输和数据共享)

    常见协议由 TCP和UDP

    • 网络层 :

    负责为分组交换网上的不同终端提供通信服务(网络数据包的传送)

    • 网络接口层 :

    提供链路管理错误检测、对不同通信媒介有关信息细节问题进行有效处理

    # 三.HTTP

    特点 :

    • 支持 B/S模式
    • 简单快速
    • 灵活
    • 无连接 : 限制每次连接只处理一个氢气球
    • 无状态 : 服务器不知道客户端是什么状态,不会记录任何信息

    # 从输入URL到页面展示到底发生了什么

    总体分为以下过程 :

    1. DNS 解析
    2. TCP 连接
    3. 发送 HTTP 请求
    4. 服务器处理请求并返回 HTTP 报文
    5. 浏览器解析渲染页面
    6. 连接结束

    # HTTP 和 HTTPS有什么区别

    • 端口号不同 : http 默认是 80 , https默认443

    • URL前缀 : http://和https://

    • 安全性和资源消耗 : HTTPS较安全,但是也耗费更多的服务器资源

    • 搜索引擎优化 : 搜索引擎通常会更青睐HTTPS协议的网站,因为HTTPS能够提供更高的安全性和用户隐私保护

    # HTTP1.0与HTTP1.1的区别

    目前我们广泛使用的是HTTP1.1

    主要有以下不同 :

    • 连接方式 : HTTP1.0为短连接,HTTP1.1为长连接
    • 响应状态码 : HTTP1.1增加了响应状态码.如100(预热请求)
    • 缓存机制 : HTTP1.1增加一些缓存策略
    • 带宽 : HTTP1.1在请求头中增加了range头域,只允许请求资源的某个部分,可以充分利用带宽

    # URI 与 URL的区别

    • URI是统一资源标识符,可以唯一表示一个资源.(身份证号)
    • URL是统一资源定位符,提供该资源的路径.(家庭住址)

    # Cookie 和Session的区别

    Cookie 和 Session 都是用来跟踪浏览器用户身份的会话方式，但是两者的应用场景不太一样。

    Cookie 存放在客户端，一般用来保存用户信息。比如token

    **Session 的主要作用就是通过服务端记录用户的状态。**比如购物车

    # 四.PING

    # PING命令的作用

    PING 命令是一种常用的网络诊断工具，经常用来测试网络中主机之间的连通性和网络延迟。

    # 五.DNS

    DNS（Domain Name System）域名管理系统，是当用户使用浏览器访问网址之后，使用的第一个重要协议。

    DNS 要解决的是域名和 IP 地址的映射问题。

    # 六.TCP 与 UDP

    # TCP 与 UDP 的区别

    首先都属于OSI传输层协议

    区别 :

    1. 是否面向连接：UDP 在传送数据之前不需要先建立连接,(如写信)。而 TCP 提供面向连接的服务，在传送数据之前必须先建立连接，数据传送结束后要释放连接(如打电话)。
    2. 是否是可靠传输：远地主机在收到 UDP 报文后，不需要给出任何确认，并且不保证数据不丢失，不保证是否顺序到达。TCP 提供可靠的传输服务，TCP 在传递数据之前，会有三次握手来建立连接，而且在数据传递时，有确认、窗口、重传、拥塞控制机制。通过 TCP 连接传输的数据，无差错、不丢失、不重复、并且按序到达。
    3. 是否有状态：这个和上面的“是否可靠传输”相对应。TCP 传输是有状态的，这个有状态说的是 TCP 会去记录自己发送消息的状态比如消息是否发送了、是否被接收了等等。为此 ，TCP 需要维持复杂的连接状态表。而 UDP 是无状态服务，简单来说就是不管发出去之后的事情了。
    4. 传输效率：由于使用 TCP 进行传输的时候多了连接、确认、重传等机制，所以 TCP 的传输效率要比 UDP 低很多。

    # TCP 三次握手和四次挥手

    # 建立连接-三次握手

    • 一次握手 : 客户端发送SYN包到服务端,然后客户端进入SYN_SEND状态
    • 二次握手 : 服务端发送SYN + ACK包到客户端并竟然与SYN_RECV状态
    • 三次握手 : 客户端发送ACK包到服务端,然后客户端和服务端都进入ESTABLISHED状态,完成TCP三次握手,进入传输状态

    # 为什么三次握手

    三次握手能确保双方收发功能都正常

    # 断开连接-四次挥手

    • 一次挥手 : 客户端发送一个FIN包到服务端,客户端进入 FIN_WAIT-1状态
    • 二次挥手 : 服务端收到FIN,发送ACK到客户端,进入CLOSE_WAIT状态,客户端进入FIN_WAIT_2状态
    • 三次挥手 : 服务端关闭与客户端的连接并发送一个FIN包,客户端请求关闭连接,服务端进入LAST_ACK状态
    • 四次挥手 : 客户端发送ACK包到服务端并进入TIME_WAIT状态,服务端静茹CLOSE状态.关闭连接

    # TCP 如何保证传输的可靠性？

    1. 基于数据块传输
    2. 对乱序的数据包重新排序以及去重
    3. 检测变化
    4. 超时重传
    5. 流量控制
    6. 拥塞控制

    # 七.IP

    属于网络层的协议，主要作用是定义数据包的格式、对数据包进行路由和寻址，以便它们可以跨网络传播并到达正确的目的地。

    # 什么是IP地址

    每个连入互联网的设备或域（如计算机、服务器、路由器等）都被分配一个 IP 地址

    当网络设备发送 IP 数据包时，数据包中包含了 源 IP 地址 和 目的 IP 地址 。

    网络设备根据目的 IP 地址来判断数据包的目的地，并将数据包转发到正确的目的地网络或子网络，从而实现了设备间的通信。

    IP 地址的唯一性和全局性保证了网络中的每个设备都可以通过其独特的 IP 地址进行标识和寻址。

    # IPv4和IPv6有什么区别

    两者都是IP协议.IPv6比较新,用来取代现在的IPv4.

    IPv6有以下优点:

    • 可用地址较多

    • 简化了配置

    • 提高了性能
