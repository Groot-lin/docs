    # 一.计算机网络体系结构

    我们知道

    计算机网络是分层的

    接下来概览全局

    看看计算机网路的体系结构

    # 1.OSI七层模型

    这是你OSI体系结构

    但是它失败了(没有占到市场)

    原因:

    1. 缺乏商业驱动
    2. 实现复杂,运行效率低
    3. 指定周期长
    4. 层次划分不太合理

    看看TCP/IP协议体系结构

    # 2.TCP/IP体系结构

    相当于将物理层和数据链路层合并成了网络接口层

    并删去了会话层和表示层

    TCP/IP协议就是占有了绝大部分市场份额体系结构

    为了方便理解

    我们还是将网络接口层分开去理解

    下面说说每层的作用:

    分层 作用
    应用层 定义进程间的交互规则
    运输层 提供通信传输服务
    网络层 提供主机间通信服务
    数据链路层 解决分组在一段链路上的问题
    物理层 解决使用何种信号传输比特流

    # 二.应用进程基于网络的通信

    透过现象看本质

    两台主机之间的网络通信实则是两个进程之间的通信

    那具体是如何通信的呢?

    假设现在Chrome浏览器中访问百度

    输入网址后

    1. 应用层

      构建HTTP报文,向下交付

      HTTP

    2. 运输层

      添加一个首部,使其成为TCP报文段

      作用是区分应用进程以及实现可靠传输

      HTTP TCP

    3. 网络层

      添加一个IP首部,使其成为IP数据报

      作用是使其可以在互联网上传输(可以被路由器转发)

      HTTP TCP IP

    4. 数据链路层

      添加一个首部和一个尾部,使其成为帧

      作用是让帧可以在一段链路或者网络上传输

      ETH HTTP TCP IP ETH

    5. 物理层

      将帧转化为转化为比特流,并添加一个前导码

    到这里可以进行传输了

    在网络中的传输只传输到网络层

    到达目的主机后再一层层拆解,最终目的主机获取应用层HTTP报文

    # 三.从输入网址到网页显示

    你一定很好奇(我猜的)

    一输入网址会回车,为啥就显示网页了呢

    上面已经说过了

    那我们看看需要涉及哪些协议吧

    # 1.HTTP

    浏览器第一步,解析URL

    URL中包含了请求服务器中的文件资源

    解析这些资源,生成HTTP报文

    # 2.DNS

    邮差拿到信件,要确定送到哪去

    DNS的作用是查询服务器域名对应的目的IP地址

    DNS是用来保存web服务器域名与IP的对应关系

    # 3.TCP

    进程间的数据传输,我把我这个进程的信息封装在TCP中

    # 4.IP

    封装自身的IP

    # 5.ARP

    每台主机都有一个mac物理地址

    mac地址与IP地址也有一层映射关系

    这就是ARP的功能了

    # 6.网卡

    网络包只是存放在内存中的一串二进制数字信息，没有办法直接发送给对方

    因此，我们需要将数字信息转换为电信号，才能在网线上传输

    负责执行这一操作的是网卡，要控制网卡还需要靠网卡驱动程序

    # 7.路由选择协议

    数据在网络中传输,一定涉及到选择哪个路由器进行转发,这就涉及到

    内部网关协议的选择问题

    如RIP,OSPF路由选择协议,选择进行转发
